# Task 001: Set up Clean Architecture Folder Structure

## Description

Create the complete clean architecture folder structure for the Trackwise app. This establishes the foundation for all future development by organizing code into clear layers (Presentation, Domain, Data) with feature-based modules.

## Acceptance Criteria

- [ ] Core folder structure created with `core/` directory containing:
  - [ ] `core/error/` (failures.dart, exceptions.dart)
  - [ ] `core/usecases/` (usecase.dart base class)
  - [ ] `core/utils/` (constants.dart, validators.dart, bluetooth_constants.dart)
  - [ ] `core/di/` (injection.dart for GetIt setup)
  - [ ] `core/theme/` (app_theme.dart, app_colors.dart)
- [ ] Features folder structure created with `features/` directory containing subdirectories for:
  - [ ] `auth/` (data, domain, presentation)
  - [ ] `items/` (data, domain, presentation)
  - [ ] `bluetooth/` (data, domain, presentation)
  - [ ] `events/` (data, domain, presentation)
  - [ ] `export/` (data, domain, presentation)
  - [ ] `profile/` (data, domain, presentation)
- [ ] Each feature has three subdirectories:
  - [ ] `data/` (models, datasources, repositories)
  - [ ] `domain/` (entities, repositories, usecases)
  - [ ] `presentation/` (bloc, pages, widgets)
- [ ] Root files created: `main.dart`, `app.dart`
- [ ] Folder structure documented in README.md

**Implementation Steps**:
1. Create `core/` directory and subdirectories
2. Create `features/` directory
3. For each feature, create three-layer structure (data, domain, presentation)
4. Create placeholder .gitkeep files in empty directories
5. Update README.md with architecture overview
