# Task 039: Set Up App Store Accounts and Internal Testing

## Description

Create Google Play Developer and Apple Developer accounts, configure internal testing tracks, and upload builds for internal testing. Invite testers and collect feedback.

## Acceptance Criteria

- [ ] Google Play Developer account created ($25 one-time)
- [ ] Apple Developer account created ($99/year)
- [ ] Google Play Internal Testing track configured
- [ ] TestFlight Internal Testing configured
- [ ] Release builds uploaded to both platforms
- [ ] 5-10 testers invited
- [ ] Testing period completed (3-5 days)
- [ ] Feedback collected and documented

## Technical Details

**Google Play Setup**:

1. **Create Account**:
   - Go to play.google.com/console
   - Pay $25 one-time registration fee
   - Complete account setup

2. **Create App**:
   - Click "Create app"
   - Name: Trackwise
   - Default language: English
   - Type: App
   - Free or paid: Free

3. **Configure Internal Testing**:
   - Go to "Testing" > "Internal testing"
   - Create release
   - Upload AAB file
   - Release name: v1.0.0
   - Release notes: "Initial internal testing release"

4. **Add Testers**:
   - Create email list
   - Add 5-10 testers
   - Share testing link

**Apple App Store Setup**:

1. **Create Account**:
   - Go to developer.apple.com
   - Enroll in Apple Developer Program ($99/year)
   - Complete enrollment (may take 24-48 hours)

2. **Create App in App Store Connect**:
   - Go to appstoreconnect.apple.com
   - Click "+" to create new app
   - Name: Trackwise
   - Bundle ID: com.yourcompany.trackwise
   - SKU: trackwise-001

3. **Configure TestFlight**:
   - In Xcode, archive app
   - Upload to App Store Connect
   - Go to TestFlight tab
   - Select build for testing
   - Add internal testers (up to 100)

4. **Add Testers**:
   - Add 5-10 internal testers
   - Send TestFlight invitations

**Build and Upload**:

```bash
# Create release keystore (if not exists)

# Add key.properties

# Build AAB
flutter build appbundle --release

# Upload to Google Play Console
# Via web interface or using fastlane
```
